# 🌟 Top Marketing - نظام إدارة التسويق الشامل

## 🛠️ التقنيات المستخدمة

- **Frontend**: Next.js 15, React 18, TypeScript
- **Styling**: Tailwind CSS, Framer Motion
- **Backend**: Supabase (Database + Auth + Storage)
- **Deployment**: Vercel
- **Version Control**: Git + GitHub

## 🚀 التثبيت والتشغيل

### المتطلبات الأساسية
- Node.js 18+ 
- npm أو yarn
- Git

### خطوات التثبيت

1. **استنساخ المشروع**
```bash
git clone https://github.com/YOUR_USERNAME/top-marketing-website.git
cd top-marketing-website
```

2. **تثبيت المكتبات**
```bash
npm install
```

3. **إعداد متغيرات البيئة**
```bash
cp .env.example .env.local
# قم بتحديث القيم في .env.local
```

4. **تشغيل المشروع**
```bash
npm run dev
```

5. **فتح المتصفح**
```
http://localhost:3000
```

## 🔧 إصلاح مشاكل Vercel

### المشاكل الشائعة وحلولها:

#### 1. مشكلة "Next.js version not detected"
```bash
# تأكد من وجود Next.js في dependencies
npm install next@14.0.4 --save

# تأكد من وجود ملف .nvmrc
echo "18.18.0" > .nvmrc
```

#### 2. مشكلة Build Errors
- تأكد من عدم وجود ملف vercel.json (احذفه إذا كان موجود)
- تأكد من وجود مجلد src/app
- تأكد من صحة package.json

#### 3. مشكلة Environment Variables
- أضف جميع المتغيرات في Vercel Dashboard
- تأكد من NEXT_PUBLIC_ prefix للمتغيرات العامة

## 📊 قاعدة البيانات Supabase

### الجداول الرئيسية:
- `users` - بيانات المستخدمين والعملاء
- `services` - الخدمات الرئيسية
- `sub_services` - الخدمات الفرعية مع الأسعار
- `orders` - طلبات العملاء
- `packages` - باقات الخدمات
- `subscriptions` - اشتراكات العملاء
- `receipts` - إيصالات الدفع
- `properties` - بيانات العقارات
- `admins` - حسابات المديرين مع الصلاحيات

### إعداد RLS Policies:
```sql
-- تمكين RLS على جميع الجداول
ALTER TABLE users ENABLE ROW LEVEL SECURITY;
ALTER TABLE orders ENABLE ROW LEVEL SECURITY;
ALTER TABLE receipts ENABLE ROW LEVEL SECURITY;

-- سياسات الأمان للمستخدمين
CREATE POLICY "Users can view own data" ON users
FOR SELECT USING (auth.uid() = id);
```
